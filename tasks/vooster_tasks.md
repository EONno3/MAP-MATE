## 실사용 프롬프트 검증 파이프라인 구축
- 실제 사용자 프롬프트 수집을 위한 설문/로그 정제 스크립트 작성
- `tmp_prompt_probe.py`에 실사용 프롬프트 데이터셋 로더 추가
- 프롬프트 세트별 기대 룸 타입 비율 메타데이터 정의
- 실행 결과를 JSON/CSV로 출력하는 요약 리포트 생성기 구현
- 목표 대비 오차를 자동 계산하는 통계 유틸 추가
- 실패 케이스 로그를 슬랙/이메일로 전달하는 알림 훅 연동
- 주기 실행을 위한 GitHub Actions 혹은 Windows 스케줄러 스크립트 작성
- 결과 리포트를 시각화(그래프/SVG)하는 보조 툴 초안 작성
- 검증 파이프라인 실행 시간을 모니터링하는 타임 트래커 도입
- 파이프라인 실행 실패 시 재시도 및 에러 스택을 수집하는 예외 처리 보강

## 보스·세이브·보물 룸 배치 제약 로직 고도화
- 존 깊이 기준으로 필수 위치 제약 조건을 정의하는 매핑 작성
- 플레이 경로 그래프에서 주요 이벤트 노드를 추출하는 헬퍼 추가
- 보스 룸과 세이브 룸 간 최소 거리/존 차이를 계산하는 검증 로직 구현
- 보물 룸 배치 시 접근 가능한 능력/락 조건을 확인하는 체크 함수 추가
- 룸 타입 할당 전 사전 유효성 검사를 수행하는 `pre_assign` 훅 도입
- 제약 위반 시 자동 재배치를 시도하는 회복 전략 구현
- 제약 충돌이 해결되지 않을 때 경고 및 수동 조정 제안 로그 생성
- 특수 룸 시각화를 위한 테스트용 SVG 그래프 렌더러 작성
- 제약 관련 파라미터를 설정 파일/Env로 외부화해 실험 가능하도록 조정
- 주요 제약 위반 사례를 포함한 회귀 테스트 데이터셋 추가

## DraftGraphView 멀티 선택 & Undo/Redo UX 개선
- 멀티 선택 상태를 추적하는 새로운 상태 관리 스토어 설계
- 드래그/리사이즈 시 그룹 행동을 적용하는 로직 분리
- 영역 선택(드래그 박스) 인터랙션 및 히트 테스트 구현
- 움직임/스냅 시 시각적 가이드라인(그리드/경계) 오버레이 추가
- Undo/Redo용 기록 스택 구조와 제한 용량 정책 정의
- 키보드 단축키 매핑(`Ctrl+Z`,`Ctrl+Shift+Z`,`Shift+드래그` 등) 연동
- 상태 변경 시 자동 저장과의 충돌을 방지하는 디바운스 조정
- 멀티 선택 상태에서 컨텍스트 메뉴 및 속성 패널 동작 재정의
- 상호작용 E2E 테스트(멀티 선택→이동→Undo→Redo) 스크립트 작성
- UX 변화에 대한 사용자 도움말/툴팁/온보딩 모달 업데이트

## TileEditor ↔ 백엔드 연동 및 Prisma 확장
- `schema.prisma`에 존 타일 데이터를 위한 모델 구조 설계
- 새 모델에 대한 Prisma 마이그레이션 생성 및 적용 검증
- `/zones/:zoneId/tiles` GET/PUT API 스펙 정의 및 라우트 구현
- 저장 시 데이터 유효성(Zod 스키마) 및 사이즈 제한 검증 추가
- TileEditor 내 로컬 상태와 서버 동기화를 위한 React Query/Zustand 연동
- 변경 감지 및 저장 버튼/자동 저장 UX 설계
- 대용량 타일 데이터 전송을 위한 압축/부분 업데이트 전략 검토
- 백엔드 단위 테스트(리포지토리/서비스/컨트롤러) 추가
- 프론트 통합 테스트(저장→재로드 검증) 작성
- 문서화: API 스펙, 타일 구조, 사용 시나리오 정리

## 낙관적 잠금과 저장 실패 처리
- 백엔드 `PUT /projects/:id/draft`에 ETag 또는 `updatedAt` 기반 버전 체크 추가
- Prisma 저장 시 낙관적 잠금 실패를 감지하는 예외 처리 삽입
- 프론트 자동 저장 흐름에 서버 버전 필드를 포함하도록 수정
- 저장 실패 시 사용자에게 선택 옵션(덮어쓰기/로컬 유지) 제공하는 모달 구현
- 재시도 시 exponential backoff 전략 및 최대 시도 횟수 설정
- 동시 편집 충돌 상태를 시뮬레이션하는 통합 테스트 작성
- 저장 실패 로그를 Sentry/로그 수집 시스템에 전송하도록 연동
- 오프라인/네트워크 오류와 버전 충돌을 구분해 메시지 개선
- 자동 저장 히스토리를 로컬에 보관해 롤백 기능 지원
- 관련 UX를 사용자 가이드/Tooltips/Doc에 반영
